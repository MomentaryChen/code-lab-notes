# Data Model: 001-design-pattern-singleton

**Feature**: 設計模式主軸與 Singleton 頁面  
**Date**: 2025-02-13

本 feature 為靜態內容頁，無後端儲存或 API；此處描述「內容結構」與「導覽實體」，供實作頁面與路由時對齊。

## 內容與導覽結構

### 主軸（Pillar）

- **用途**：首頁上的一個主題區塊。
- **屬性**：標題、描述、進入連結（path）。
- **本 feature 變更**：原「Java OOP」主軸改為「設計模式」；標題與描述改為設計模式相關，連結改為 `/design-pattern`。

### 設計模式區塊首頁（Hub）

- **用途**：設計模式主軸的落地頁，列出子主題。
- **屬性**：標題、副標題、子主題列表（至少一項：Singleton）；每個子主題含名稱與連結。
- **關係**：由首頁主軸連結進入；可連結至各設計模式子頁與返回首頁。

### Singleton 專頁

- **用途**：單一設計模式說明頁，靜態內容。
- **內容區塊**（邏輯結構，非資料庫）：
  1. **解釋**：單例意涵、為何需要、適用情境、與其他模式的區別。
  2. **使用介紹**：何時使用、概念性使用方式、注意事項或常見陷阱。
  3. **範例**（可選）：以 Java / Spring Boot 為例的程式碼片段，支援說明。
- **屬性**：標題、導覽（返回設計模式 hub、返回首頁）。

## 路由對應

| 路由 | 對應實體 | 說明 |
|------|----------|------|
| `/` | 首頁 | 主軸區塊含「設計模式」卡片，連結至 `/design-pattern` |
| `/design-pattern` | 設計模式 hub | 列出子主題，含 Singleton 連結 |
| `/design-pattern/singleton` | Singleton 專頁 | 說明與使用介紹 |
| `/oop` | 重新導向 | 導向 `/design-pattern`，相容舊書籤 |

## 驗證規則（來自規格）

- 所有對使用者可見文字為正體中文（FR-006）。
- 設計模式區塊內頁面須提供返回首頁或返回設計模式區塊的導覽（FR-007）。
