貪婪演算法（Greedy Algorithms）

貪婪演算法的核心精神是：在每一步都做出當前看起來「最好」的選擇，希望最後能得到全域最佳解。 並非所有問題都適合貪婪策略，但對結構良好的問題，貪婪法可以在保證正確性的前提下，大幅簡化實作。

活動選擇問題（Interval Scheduling）

給定一組活動，每個活動都有開始時間與結束時間，且在同一時間只能進行一個活動。 問題是：最多可以選出多少個互不重疊的活動？

  • 貪婪思想：每一步都嘗試「為後面留下最多空間」。
  • 正確策略：永遠優先選擇「結束時間最早」且與已選活動不衝突的活動。
  • 關鍵性質：此問題同時滿足「貪婪選擇性」與「最優子結構」，因此貪婪法可保證得到最大可選活動數量。
互動示例：以最早結束時間為優先的貪婪選擇
橫軸為時間（單位：小時） 綠色：被選中的活動　紅色：被捨棄的活動
點擊「開始貪婪選擇」觀看演算法如何一步步挑選活動。
// 以「結束時間最早」作為貪婪選擇策略
function selectActivities(activities) {
  // 依結束時間排序
  activities.sort((a, b) => a.end - b.end);
  const selected = [];
  let lastEnd = -Infinity;

  for (const act of activities) {
    if (act.start >= lastEnd) {
      selected.push(act);
      lastEnd = act.end;
    }
  }
  return selected;
}
